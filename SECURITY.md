# Security Policy

## Reporting a Vulnerability

Report security vulnerabilities through GitHub private vulnerability reporting
for `clicksopendoors/burpvalve`:

https://github.com/clicksopendoors/burpvalve/security

Do not open a public issue with exploit details.

This repository is being prepared for public launch.
GitHub Private Vulnerability Reporting activation and working-channel
verification are deferred to the visibility-flip gate. The launch gate must
confirm that the GitHub security reporting channel is active and reachable before
the repository is made public.

## Supported Versions

The current released version is the supported version for security reports.
Older releases may receive fixes when the maintainer decides the risk and patch
scope justify a backport.

## Disclosure

Coordinate disclosure after triage and fix handling. The maintainer will
publish concrete status when there is a useful update to share.
